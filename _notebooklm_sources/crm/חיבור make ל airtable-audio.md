# חיבור make ל airtable-audio.mp4

🎯 מטרת השיעור

המטרה של השיעור היא ללמוד כיצד יוצרים חיבור בין Make ל־Airtable, להבין את שני סוגי החיבורים האפשריים — OAuth ו־API Key — ולבצע בפועל יצירת Connection חדש מתוך הסנאריו כדי לאפשר עבודה מול טבלאות Airtable במודולים השונים.

📌 נקודות מרכזיות

החיבור תמיד מתחיל מתוך הסנאריו במייק

יוצרים Scenario חדש.

מחפשים את Airtable.

בוחרים מודול שאינו Watch (למשל: Search Records), כי Watch לרוב דורש Webhook ולא חיבור ישיר.

במודול יופיע כפתור Create a Connection / Add ליצירת חיבור חדש.

שני סוגי החיבורים: OAuth לעומת API Key

OAuth

מאפשר התחברות עם שם משתמש וסיסמה.

פותח חלון חיצוני של Airtable עם מסך הרשאות.

פשוט ונוח לשימוש.

API Key

משתמש ב־Token שמוגדר ב־Airtable.

דורש שליחה ידנית של ה־Token למייק.

משמש כאשר רוצים שליטה מלאה בהרשאות ע"י מערכת הטוקנים של Airtable.

שני החיבורים מתפקדים כמעט אותו דבר, אך API Key מאפשר רמות הרשאה גמישות יותר.

יצירת חיבור מסוג OAuth

בוחרים “Create a Connection”.

נותנים שם, לדוגמה: test-oauth.

לוחצים Save -> נפתח חלון התחברות ל־Airtable.

מאשרים הרשאות (Scopes) ולוחצים Grant Access.

לאחר האישור החיבור מוכן ומופיע ברשימת החיבורים.

יצירת חיבור באמצעות API Token

בוחרים באפשרות Token בתוך Make.

נותנים שם לחיבור, לדוגמה: test-api-key.

Make מבקשת את ה־Token.

להשגת ה־Token:

נכנסים ל־Airtable -> לוחצים על אייקון המשתמש.

נכנסים ל־Builder Hub.

בוחרים Personal Access Tokens.

יוצרים Token חדש -> נותנים שם -> מסמנים הרשאות (Scopes).

לוחצים Create token ומעתיקים את המפתח.

לאחר מכן מדביקים את ה־Token במייק ולוחצים Save.

לאחר יצירת החיבור — ניתן לעבוד עם כל מודולי Airtable

מרגע שהחיבור תקין, ניתן להשתמש במודולים כמו:

Search Records

Create Record

Update Record

Delete Record

Watch Records (טריגר עם Webhook)

החיבור מופיע ב־Dropdown של המודולים וניתן לעשות בו שימוש חוזר בכל סנאריו.

📝 סיכום השיעור

השיעור מדגים כיצד מחברים את Make ל־Airtable בעזרת OAuth או API Key, החל מבחירת מודול, דרך יצירת Connection חדש ועד שליפת הטוקן מתוך Airtable. לאחר יצירת החיבור ניתן לבנות אוטומציות מלאות המבצעות קריאה ועדכון של נתונים בתוך בסיסי Airtable.
