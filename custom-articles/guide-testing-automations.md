# מדריך: בדיקת אוטומציות לפני העלייה לאוויר

## למה חשוב לבדוק?

אוטומציה שלא נבדקה יכולה:
- לשלוח הודעות שגויות ללקוחות
- ליצור רשומות כפולות ב-CRM
- לצרוך אופרציות מיותרות (=עלות כספית)
- לחסום את החשבון שלכם בפלטפורמות

## שלב 1: בדיקה ידנית — Run Once

### מה זה Run Once?
ב-Make.com, לחצו על **"Run once"** (כפתור בפינה השמאלית התחתונה) כדי להריץ את הסנריו פעם אחת ולראות מה קורה.

### איך לבדוק
1. הכינו **נתוני בדיקה** — מספר טלפון שלכם, אימייל בדיקה
2. לחצו **Run Once**
3. שלחו את הטריגר (טופס, הודעה, וכו')
4. עקבו אחרי **כל מודול** — לחצו על כל בועה לראות Input/Output
5. ודאו שהנתונים עוברים נכון מול מודול למודול

### מה לבדוק
- ✅ האם הנתונים מגיעים מהטריגר?
- ✅ האם השדות ממופים נכון?
- ✅ האם ההודעה/אימייל/רשומה נוצרת כמצופה?
- ✅ האם יש שגיאות (עיגול אדום על מודול)?

## שלב 2: בדיקת מקרי קצה

### מה לבדוק
1. **שדות ריקים** — מה קורה אם הלקוח לא מילא טלפון?
2. **פורמט שגוי** — מספר טלפון עם מקפים, אימייל בלי @
3. **תווים מיוחדים** — שמות עם גרש (כמו "דניאל'ה"), אמוג'ים
4. **טקסט ארוך** — הודעה עם 1000 תווים — האם נחתכת?

### איך להגן
- **הוסיפו פילטרים** — Filter אחרי הטריגר: `phone is not empty`
- **נקו נתונים** — השתמשו ב-Set Variable לניקוי פורמט
- **הוסיפו Error Handler** — תפסו שגיאות ושלחו התראה לעצמכם

## שלב 3: בדיקת Error Handling

### מה זה Error Handler?
מודול שתופס שגיאות ומחליט מה לעשות:
- **Ignore** — התעלם מהשגיאה והמשך
- **Retry** — נסה שוב (מומלץ לשגיאות רשת)
- **Break** — עצור את הסנריו ושלח התראה
- **Rollback** — בטל את כל הפעולות

### איך להוסיף
1. לחצו **ימני** על מודול
2. בחרו **"Add error handler"**
3. בחרו את סוג הטיפול
4. מומלץ: Retry עם 3 ניסיונות ומרווח של 60 שניות

## שלב 4: בדיקת עומסים

### למה זה חשוב
אם יש לכם דף נחיתה שמקבל 100 לידים בשעה, האוטומציה צריכה לעמוד בזה.

### מה לבדוק
- **Rate limits** — כל API מגביל כמות בקשות (ראו מדריך פתרון בעיות חיבור)
- **Queue** — Make.com מנהל תור — אם יש יותר מדי executions, הם ימתינו
- **אופרציות** — ודאו שיש לכם מספיק אופרציות בתוכנית

### איך לחשב אופרציות
- כל מודול = 1 אופרציה
- סנריו עם 5 מודולים × 100 הרצות ביום = 500 אופרציות
- תוכנית Free: 1,000 אופרציות/חודש
- תוכנית Core: 10,000 אופרציות/חודש

## שלב 5: העלאה לאוויר

### רשימת בדיקות לפני הפעלה
1. ✅ הרצת Run Once עם נתוני בדיקה — עבר בהצלחה
2. ✅ בדיקת שדות ריקים — יש פילטרים
3. ✅ בדיקת Error Handler — מוגדר בכל מודול קריטי
4. ✅ חישוב אופרציות — מספיק בתוכנית
5. ✅ בדיקת הודעות — הטקסט תקין, בלי שגיאות כתיב
6. ✅ בדיקת תזמון — הסנריו מוגדר לרוץ בתדירות הנכונה

### הפעלה
1. לחצו על **Toggle** להפעלת הסנריו (ON)
2. הגדירו תדירות (Immediately, Every 15 min, וכו')
3. **שימו לב**: Immediately = הסנריו מאזין כל הזמן (צורך יותר אופרציות)

## שלב 6: מעקב אחרי העלאה

### ב-Make.com
- **Execution History** — בדקו כל יום בשבוע הראשון
- **Error Notifications** — הפעילו ב-Settings → Notifications
- **Incomplete Executions** — בדקו שאין הרצות שנתקעו

## טיפים חשובים
- **תמיד בדקו עם המספר שלכם קודם** — לא עם מספר של לקוח
- **שמרו גרסה עובדת** — לפני שינויים, שכפלו את הסנריו (Clone)
- **התחילו עם תדירות נמוכה** — Every 15 min ולא Immediately
- **תעדו את הסנריו** — הוסיפו Notes לכל מודול שמסביר מה הוא עושה

## שגיאות נפוצות
- **"Scenario is not active"** — שכחתם להפעיל. לחצו ON
- **"No data to process"** — הטריגר לא קיבל נתונים. ודאו שהחיבור עובד
- **"Operation limit reached"** — נגמרו האופרציות. שדרגו תוכנית או המתינו לחודש הבא
- **"Incomplete execution"** — הסנריו נתקע באמצע. בדקו את המודול האחרון שרץ
