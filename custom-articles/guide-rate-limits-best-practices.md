# מדריך: מגבלות שליחה ושיטות עבודה מומלצות

## מהם Rate Limits?

Rate Limits (מגבלות קצב) הם הגבלות שכל שירות מפעיל כדי למנוע עומס. אם שולחים יותר מדי בקשות — השירות חוסם את הגישה זמנית (או קבוע!).

## מגבלות לפי שירות

### WhatsApp (GreenAPI)
- **הודעות**: עד 50 הודעות בדקה
- **קבוצות**: עד 5 קבוצות ביום (יצירה)
- **חסימה**: שליחת ספאם = חסימת מספר לצמיתות
- **המלצה**: לא יותר מ-200 הודעות בשעה

### Facebook API
- **קריאות**: 200 קריאות לשעה לכל Token
- **פרסום**: 25 פוסטים ביום לכל דף
- **Lead Ads**: ללא הגבלה מיוחדת (בדרך כלל)
- **Messenger**: 250 הודעות ליום (לחדשים), ללא הגבלה אחרי Review

### Instagram API
- **קריאות**: 200 קריאות לשעה
- **פרסום**: 25 פוסטים ב-24 שעות
- **DM**: 1,000 הודעות ביום (Business account)
- **תגובות**: 60 תגובות לשעה

### Google Sheets
- **קריאות**: 100 בקשות ל-100 שניות
- **כתיבה**: 100 בקשות ל-100 שניות
- **טבלה**: עד 10 מיליון תאים

### Monday.com
- **קריאות**: 5,000 ליום (Pro), 10,000 ליום (Enterprise)
- **Mutations**: 10,000 ליום
- **Complexity**: 1,000,000 נקודות complexity ליום

### OpenAI (ChatGPT)
- **RPM**: 60-10,000 בקשות לדקה (תלוי בתוכנית)
- **TPM**: 60,000-2,000,000 tokens לדקה
- **תמונות**: 50 תמונות לדקה (DALL-E)

### Gmail
- **שליחה**: 500 הודעות ביום (חשבון רגיל), 2,000 ביום (Workspace)
- **נמענים**: 500 נמענים ביום
- **גודל**: עד 25MB לכל הודעה

## כלל אצבע: 70% מהמקסימום

לעולם אל תתכננו לעבוד ב-100% מהמגבלה. **השתמשו ב-70%** כמקסימום בפועל:
- אם המגבלה 200/שעה → תכננו ל-140/שעה
- אם המגבלה 500/יום → תכננו ל-350/יום

## איך למנוע חסימות ב-Make.com

### 1. הוסיפו השהיה בין הודעות
- מודול **Tools → Sleep** — המתנה של 1-2 שניות בין הודעות
- חיוני כששולחים הודעות WhatsApp ברצף

### 2. השתמשו בפילטרים
- סננו נתונים לא רלוונטיים **לפני** שליחת API
- כל קריאת API שנחסכת = אופרציה שנחסכת

### 3. Batch Processing
- במקום לעדכן שורה-שורה ב-Google Sheets, השתמשו ב-"Add Multiple Rows"
- במקום 100 קריאות API → קריאה אחת עם 100 שורות

### 4. Error Handler עם Retry
- הוסיפו Error Handler לכל מודול שקורא ל-API
- הגדירו Retry: 3 ניסיונות, מרווח 60 שניות
- זה תופס שגיאות זמניות (429, 500, timeout)

### 5. תזמון חכם
- **אל תריצו כל דקה** אם כל 15 דקות מספיק
- **הפיצו עומסים** — אם יש 3 סנריואים, תזמנו בשעות שונות
- **שעות שקטות** — העדיפו שליחה בשעות 8:00-20:00 (פחות עומס)

## WhatsApp: מניעת חסימת מספר

### כללים קריטיים
1. **בקשו הסכמה** — תמיד ודאו שהנמען הסכים לקבל הודעות
2. **תנו אפשרות ביטול** — "שלח 'הפסק' להסרה מרשימת התפוצה"
3. **אל תשלחו לאנשים שלא מכירים אתכם** — זה ספאם
4. **הגבילו תדירות** — לא יותר מ-2-3 הודעות ביום לאותו אדם
5. **גוונו תוכן** — אל תשלחו את אותה הודעה בדיוק ל-1000 אנשים

### סימנים שהמספר עלול להיחסם
- אנשים חוסמים את המספר שלכם
- הודעות לא מגיעות (שני V אפורים)
- WhatsApp שולח אזהרה

## טיפים חשובים
- **תעדו את המגבלות** של כל שירות שאתם משתמשים בו
- **הגדירו התראות** ב-Make.com כשסנריו נכשל — כך תדעו מיד אם נחסמתם
- **תתחילו קטן** — 10 הודעות ביום, ואז 50, ואז 100
- **עקבו אחרי אופרציות** — ב-Make.com Dashboard ← Operations counter

## שגיאות נפוצות
- **"Error 429 Too Many Requests"** — עברתם את המגבלה. המתינו ונסו שוב
- **"WhatsApp number banned"** — שלחתם ספאם. פנו לתמיכה של WhatsApp (קשה לשחזר)
- **"Facebook rate limit hit"** — צמצמו את תדירות הסנריו
- **"Gmail sending limit exceeded"** — המתינו 24 שעות או שדרגו ל-Google Workspace
- **"OpenAI quota exceeded"** — שדרגו תוכנית או המתינו לאיפוס החודשי
