# מתכון אוטומציה: עדכון ב-Monday.com → הודעת WhatsApp

## תיאור התהליך
אוטומציה שמזהה שינוי סטטוס או עדכון בבורד ב-Monday.com ושולחת הודעת WhatsApp אוטומטית לאיש הקשר הרלוונטי — למשל עדכון ללקוח שהמשימה שלו הושלמה, או התראה לצוות.

## מודולים נדרשים ב-Make.com
- **Monday.com — Watch Board Changes**: טריגר שמזהה שינויים בבורד
- **Monday.com — Get Item** (אופציונלי): שליפת פרטים נוספים
- **HTTP — Make a Request**: שליחת WhatsApp דרך GreenAPI

## חיבורים מוקדמים
1. חיבור Monday.com: הגדרות → Connections → Monday.com → API Token מתוך Monday (Admin → API)
2. חיבור GreenAPI: Instance ID + API Token

## שלבים לבניית הסנריו

### שלב 1: טריגר — Monday.com
1. צרו סנריו חדש ב-Make.com
2. הוסיפו מודול **Monday.com → Watch Board Changes**
3. בחרו את הבורד הרלוונטי
4. הגדירו איזה סוג שינוי לעקוב: Change Column Value (שינוי ערך בעמודה)

### שלב 2: פילטר — רק שינויי סטטוס רלוונטיים
1. הוסיפו **Filter** אחרי הטריגר
2. תנאי: Column Title = "סטטוס" AND New Value = "הושלם"
3. כך רק שינויים רלוונטיים יפעילו את ה-WhatsApp

### שלב 3: שליפת פרטי הפריט
1. הוסיפו **Monday.com → Get an Item**
2. הזינו את ה-Item ID שהתקבל מהטריגר
3. כך תקבלו את כל העמודות: שם לקוח, טלפון, שם פרויקט וכו'

### שלב 4: שליחת WhatsApp
1. הוסיפו **HTTP → Make a request**
2. Body:
   ```json
   {
     "chatId": "{{phone}}@c.us",
     "message": "שלום {{clientName}}! 🎉\n\nרצינו לעדכן אותך שהמשימה \"{{itemName}}\" הושלמה בהצלחה.\n\nשאלות? אנחנו כאן! 💬"
   }
   ```

## וריאציות נפוצות

### התראה לצוות (לא ללקוח)
- במקום לשלוח ללקוח, שלחו לקבוצת WhatsApp של הצוות
- שנו את ה-chatId ל-Group ID (אפשר לקבל מ-GreenAPI)
- הודעה: "📋 {{assignee}} סיים את המשימה: {{itemName}}"

### שינוי סטטוס ל"דחוף"
- פילטר: New Value = "דחוף" או "בעיה"
- הודעה מיידית למנהל עם פרטי המשימה
- אפשר להוסיף Router: גם WhatsApp וגם אימייל

### עדכון תאריך יעד
- טריגר: שינוי בעמודת Date
- שליחת הודעה ללקוח: "תאריך היעד עודכן ל-{{newDate}}"

## טיפים חשובים
- Monday.com API Token לא פג תוקף, אבל אם משנים סיסמה הוא מתבטל
- השתמשו בעמודת Phone ב-Monday כדי לשמור מספרי טלפון בפורמט אחיד
- מומלץ להגביל את הטריגר לבורד ספציפי ולא לכל הבורדים — חוסך אופרציות
- הוסיפו Error Handler כדי לתפוס מספרי טלפון לא תקינים

## שגיאות נפוצות
- **טריגר מופעל על כל שינוי**: הוסיפו פילטר על שם העמודה וערך חדש
- **אין מספר טלפון בפריט**: ודאו שיש עמודת Phone בבורד ושהיא מלאה
- **שגיאת Monday API**: ודאו שה-Token שייך למשתמש Admin או Owner של הבורד
- **הודעות כפולות**: Monday שולח אירוע לכל שינוי בכל עמודה — הפילטר הכרחי
