# 💬 תגובה בפייסבוק → התראת WhatsApp מיידית

## 📝 תיאור התהליך
מישהו הגיב לכם על פוסט בדף העסקי? תקבלו על זה הודעת WhatsApp **תוך שניות** — עם שם המגיב, תוכן התגובה, ולינק לפוסט.

מתאים לעסקים שרוצים **להגיב מהר ללידים חמים** שמגיבים על פוסטים.

## 🧩 מודולים נדרשים ב-Make.com
- **Facebook Pages → Watch Comments** — מאזין לתגובות חדשות בדף העסקי
- **HTTP → Make a Request** — שולח הודעת WhatsApp דרך GreenAPI

## 🔌 חיבורים מוקדמים
1. **חיבור Facebook**: הגדרות → Connections → חפשו Facebook Pages → התחברו עם החשבון שמנהל את הדף העסקי
2. **חיבור GreenAPI**: הגדרות → Connections → חפשו HTTP → הזינו Instance ID ו-API Token מחשבון GreenAPI שלכם

## 📋 שלבים לבניית הסנריו

### שלב 1: הוספת טריגר Facebook
1. פתחו סנריו חדש ב-Make.com
2. הוסיפו מודול **Facebook Pages → Watch Comments**
3. בחרו את החיבור לפייסבוק
4. בחרו את הדף העסקי שלכם
5. ב-Where to start הגדירו מאיזה תאריך לצפות

### שלב 2: הוספת פילטר (אופציונלי)
רוצים לקבל התראות רק על תגובות מסוימות? הוסיפו פילטר:
1. לחצו על הקו שבין הטריגר למודול הבא
2. הגדירו תנאי, למשל: **Message** contains `מחיר` או `פרטים`
3. ככה רק תגובות רלוונטיות יגיעו אליכם

### שלב 3: שליחת הודעת WhatsApp
1. הוסיפו מודול **HTTP → Make a request**
2. הגדירו את הפרטים:
   - **URL:**
     ```
     https://api.green-api.com/waInstance{{InstanceID}}/sendMessage/{{APIToken}}
     ```
   - **Method:** POST
   - **Body type:** JSON
   - **גוף ההודעה:**
     ```json
     {
       "chatId": "972XXXXXXXXX@c.us",
       "message": "🔔 תגובה חדשה בפייסבוק!\n\n👤 מגיב: {{name}}\n💬 תוכן: {{message}}\n📌 פוסט: {{post_id}}"
     }
     ```

### שלב 4: הפעלה ובדיקה
1. שמרו את הסנריו
2. לחצו **Run Once**
3. כתבו תגובת בדיקה על פוסט בדף שלכם
4. בדקו שההודעה הגיעה ב-WhatsApp
5. הכל עובד? העבירו ל-**ON** והגדירו תזמון

## 🔄 חלופה: דרך ManyChat
אם יש לכם חשבון ManyChat מחובר לפייסבוק ול-WhatsApp:
1. צרו אוטומציה חדשה ב-ManyChat
2. טריגר: **Facebook Comments** → בחרו את הפוסט
3. Action: **Send WhatsApp Message**
4. כתבו את ההודעה עם משתנים דינמיים (שם המגיב וכו')

## 💡 טיפים חשובים
- ודאו שלדף הפייסבוק יש לכם **הרשאות ניהול** מלאות
- ב-GreenAPI הטלפון צריך להיות **מחובר ופעיל** כל הזמן
- מומלץ להוסיף **Error Handler** מסוג Ignore כדי שסנריו לא ייעצר בגלל שגיאה בודדת
- שולחים הודעות ללקוחות? חובה **הסכמה מראש** לפי חוק הספאם

## ⚠️ שגיאות נפוצות
- **שגיאת 401 מ-GreenAPI** — בדקו שה-Instance ID וה-API Token מעודכנים
- **לא מגיעות תגובות** — ודאו שהטריגר מוגדר לדף הנכון ושהסנריו מופעל
- **הודעה לא נשלחת** — מספר הטלפון חייב להיות בפורמט `972` בלי אפס (למשל: `972501234567`)
