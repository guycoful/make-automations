# מדריך: פתרון בעיות חיבור באוטומציות

## החיבור נכשל — מה עושים?

בעיות חיבור הן הבעיה מספר 1 באוטומציות. הנה מדריך שיטתי לאבחון ותיקון.

## בעיה 1: החיבור לא נוצר בכלל

### סימנים
- לחיצה על "Connect" לא פותחת חלון
- חלון ההתחברות נפתח אבל נסגר מיד
- הודעת שגיאה: "Connection failed"

### פתרונות
1. **נסו דפדפן אחר** — Chrome עובד הכי טוב עם Make.com
2. **בטלו חוסם פופ-אפים** — Make.com פותח חלון חדש לאימות
3. **נקו Cookies** — לפעמים עוגיות ישנות מפריעות
4. **ודאו שאתם מחוברים** לשירות (Facebook, Google) באותו דפדפן
5. **נסו Incognito mode** — מבטל תוספים שעלולים להפריע

## בעיה 2: החיבור היה עובד ופתאום הפסיק

### סיבות נפוצות
- **Token פג תוקף** — OAuth tokens (Facebook, Google) פגים אחרי 60 יום
- **שינוי סיסמה** — שינוי סיסמה בשירות מבטל את ה-Token
- **הרשאות הוסרו** — מישהו הסיר את ההרשאות מהאפליקציה
- **עדכון API** — השירות שינה את ה-API שלו

### איך לתקן
1. כנסו ל-**Make.com → Connections**
2. מצאו את החיבור עם ⚠️ (סימן אזהרה)
3. לחצו **Reauthorize**
4. אשרו מחדש את ההרשאות
5. הריצו את הסנריו ובדקו

## בעיה 3: "Error 429 — Too Many Requests"

### מה זה
השירות חוסם אתכם כי שלחתם יותר מדי בקשות.

### כמה מותר?
- **Facebook API**: 200 קריאות לשעה לכל Token
- **Google Sheets**: 100 בקשות ל-100 שניות
- **GreenAPI**: 50 הודעות לדקה
- **OpenAI**: תלוי בתוכנית (RPM - Requests Per Minute)

### פתרונות
1. **הוסיפו Sleep בין בקשות** — Tools → Sleep → 1-2 שניות
2. **צמצמו תדירות הסנריו** — במקום כל 5 דקות, הריצו כל 15 דקות
3. **השתמשו ב-Batch processing** — עבדו על כמה רשומות ביחד
4. **הוסיפו Error Handler** עם Retry — נסו שוב אחרי 60 שניות

## בעיה 4: GreenAPI מתנתק

### למה זה קורה
- הטלפון כבוי או ללא אינטרנט
- פתחתם WhatsApp Web בדפדפן אחר
- WhatsApp עדכן את האפליקציה
- עברו יותר מ-14 יום בלי פעילות

### פתרון
1. כנסו לדשבורד GreenAPI
2. בדקו סטטוס: אם "not authorized" — צריך לסרוק QR מחדש
3. בטלפון: WhatsApp → מכשירים מקושרים → ודאו שהחיבור פעיל
4. אם לא מופיע — סרקו QR חדש

### מניעה
- השאירו את הטלפון תמיד מחובר ל-WiFi
- אל תפתחו WhatsApp Web במקביל ל-GreenAPI
- בדקו את הסטטוס פעם בשבוע

## בעיה 5: Facebook/Instagram חיבור נשבר

### סיבות
- Token פג אחרי 60 יום (Long-lived token)
- שיניתם סיסמת Facebook
- הדף העסקי שונה או הועבר
- פייסבוק הסירה את האפליקציה

### פתרון
1. ב-Make.com: Connections → Facebook → **Reauthorize**
2. אם לא עוזר: **Delete** את החיבור וצרו חדש
3. ודאו שהדף העסקי עדיין פעיל ומפורסם
4. בדקו ב-Facebook: Settings → Business Integrations → שהאפליקציה קיימת

## בעיה 6: Monday.com API שגיאות

### "Authentication failed"
- API Token שגוי — העתיקו מחדש מ-Monday → Admin → API
- שימו לב: שינוי סיסמה ב-Monday מבטל את ה-Token

### "Board not found"
- הבורד נמחק, הועבר או שאין לכם הרשאות
- ודאו שה-Token שייך למשתמש שיש לו גישה לבורד

## כלים לאבחון

### ב-Make.com
- **Execution History** — לחצו על הסנריו → History → ראו פרטי שגיאה מלאים
- **Debug Mode** — הריצו Once ועקבו אחרי כל מודול
- **Connection Status** — Connections → בדקו סטטוס כל חיבור

### כלים חיצוניים
- **Postman** — בדקו API ידנית מחוץ ל-Make
- **Webhook.site** — בדקו שהנתונים מגיעים נכון

## טיפים חשובים
- **בדקו חיבורים פעם בשבוע** — אל תחכו שמשהו יישבר
- **הגדירו Error Notifications** ב-Make.com — תקבלו מייל כשסנריו נכשל
- **תעדו את ה-Tokens שלכם** — שמרו במנהל סיסמאות עם תאריך יצירה
- **חברו עם חשבון ייעודי** — לא החשבון האישי שלכם

## שגיאות נפוצות
- **"ECONNREFUSED"** — השרת של השירות לא מגיב. נסו שוב מאוחר יותר
- **"ETIMEDOUT"** — הבקשה לקחה יותר מדי זמן. בדקו אינטרנט ונסו שוב
- **"SSL Certificate Error"** — בעיית אבטחה. ודאו שאתם משתמשים ב-HTTPS
- **"Invalid JSON"** — ה-Body שהרכבתם לא תקני. בדקו סוגריים וגרשיים
