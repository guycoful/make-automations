# מדריך: שגיאות נפוצות ב-Make.com ואיך לפתור אותן

## איך לקרוא שגיאה ב-Make.com?

כשסנריו נכשל, Make.com מציג:
1. **עיגול אדום** על המודול שנכשל
2. **הודעת שגיאה** עם קוד ותיאור
3. **Bundle** — הנתונים שגרמו לשגיאה

לחצו על המודול האדום כדי לראות את פרטי השגיאה המלאים.

## שגיאות HTTP נפוצות

### 400 — Bad Request
**מה זה**: הבקשה שנשלחה לא תקינה.
**סיבות נפוצות**:
- שדה חובה חסר (למשל: שלחתם בלי chatId)
- פורמט לא נכון (JSON שבור, תו לא חוקי)
- ערך לא תקין (מספר טלפון קצר מדי)

**פתרון**: בדקו את ה-Input של המודול — לחצו על הבועה וראו מה נשלח. השוו מול התיעוד של ה-API.

### 401 — Unauthorized
**מה זה**: האימות נכשל — Token שגוי או פג תוקף.
**פתרון**: חדשו את החיבור ב-Connections → Reauthorize.

### 403 — Forbidden
**מה זה**: אין הרשאות מספיקות.
**סיבות נפוצות**:
- לא נתתם את כל ההרשאות בעת החיבור
- הדף/בורד/טבלה לא נגישים למשתמש שהתחבר
- האפליקציה לא אושרה (למשל בפייסבוק)

**פתרון**: מחקו את החיבור וצרו חדש, הפעם אשרו **את כל ההרשאות**.

### 404 — Not Found
**מה זה**: המשאב לא נמצא.
**סיבות נפוצות**:
- URL שגוי במודול HTTP
- הבורד/טבלה/דף נמחק או שונה
- ID לא נכון

**פתרון**: ודאו שה-URL/ID שהזנתם נכונים ושהמשאב עדיין קיים.

### 429 — Too Many Requests
**מה זה**: עברתם את מגבלת הבקשות.
**פתרון**: הוסיפו Sleep בין בקשות, צמצמו תדירות, או הוסיפו Error Handler עם Retry.

### 500 — Internal Server Error
**מה זה**: שגיאה בשרת של השירות החיצוני (לא אצלכם).
**פתרון**: המתינו ונסו שוב. אם חוזר — פנו לתמיכה של השירות.

## שגיאות Make.com ספציפיות

### "RuntimeError: No data to process"
**מה קרה**: הטריגר לא קיבל נתונים.
**פתרון**:
- ודאו שהסנריו מופעל (ON)
- ודאו שהטריגר מוגדר נכון (Webhook URL נכון, בורד נכון, וכו')
- שלחו נתונים מחדש ובדקו

### "IncompleteExecution"
**מה קרה**: הסנריו התחיל אבל לא הסתיים.
**סיבות**: timeout, שגיאה לא מטופלת, חוסר אופרציות.
**פתרון**: כנסו ל-Incomplete Executions → בדקו מה קרה → הריצו מחדש או מחקו.

### "OperationLimitExceeded"
**מה קרה**: נגמרו האופרציות בתוכנית.
**פתרון**: המתינו לאיפוס חודשי, שדרגו תוכנית, או ייעלו את הסנריו (פחות מודולים = פחות אופרציות).

### "ConnectionError"
**מה קרה**: Make.com לא הצליח להתחבר לשירות.
**פתרון**: בדקו חיבור אינטרנט, ודאו שהשירות זמין (בדקו ב-downdetector.com), נסו שוב.

### "DataError: Invalid JSON"
**מה קרה**: ה-JSON שהרכבתם לא תקין.
**בדקו**:
- סוגריים מסולסלים `{ }` מאוזנים
- כל מפתח בגרשיים כפולים `"key"`
- פסיקים בין שדות (בלי פסיק אחרי האחרון)
- השתמשו ב-jsonlint.com לבדיקה

## Error Handlers — איך להגדיר

### הוספת Error Handler
1. לחצו **ימני** על מודול
2. **Add error handler**
3. בחרו סוג:
   - **Resume** — המשך עם ערך ברירת מחדל
   - **Retry** — נסה שוב (הכי נפוץ)
   - **Ignore** — התעלם והמשך למודול הבא
   - **Break** — עצור ושמור ל-Incomplete Executions
   - **Rollback** — בטל הכל

### הגדרת Retry מומלצת
- **ניסיונות**: 3
- **מרווח**: 60 שניות
- **סוג שגיאות**: ConnectionError, RuntimeError

## כלי Debug

### 1. Run Once + Inspector
- הריצו **Run Once** ולחצו על כל מודול
- ראו **Input** (מה נכנס) ו-**Output** (מה יצא)
- זהו מידע שגוי

### 2. Execution History
- כנסו לסנריו → **History**
- ראו את כל ההרצות — מוצלחות וכושלות
- לחצו על הרצה כושלת לראות פרטים

### 3. DevTool של Make
- ב-Scenario Editor → לחצו על **DevTool** (פינה תחתונה)
- ראו לוגים מפורטים של כל פעולה

## טיפים חשובים
- **תמיד הוסיפו Error Handler** למודולים שקוראים ל-API חיצוני
- **הפעילו Email Notifications** ב-Make.com Settings
- **בדקו Execution History** פעם בשבוע
- **שמרו על סנריו פשוט** — ככל שיש פחות מודולים, יש פחות מקומות לשגיאות

## שגיאות נפוצות
- **"Scenario has been disabled due to errors"** — Make.com כיבה את הסנריו אחרי שגיאות חוזרות. תקנו את הבעיה והפעילו מחדש
- **"Data transfer limit reached"** — העברתם יותר מדי נתונים. שדרגו תוכנית
- **"Module timeout"** — המודול לקח יותר מ-5 דקות. ייעלו את הבקשה
- **"Webhook queue is full"** — יותר מדי אירועים ממתינים. הגבירו תדירות הסנריו
